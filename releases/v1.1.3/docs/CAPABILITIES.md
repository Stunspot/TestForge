# TestForge: capabilities

Use the [quick start](QUICK-START.md) to enter through one of these capability boundaries.

## software-verification

- construct a scope-to-release evidence chain
- inspect impact/risk/invariants/scenarios/oracles
- create or repair compatible tests and verification records
- run/normalize exact checks and issue one evidence-bounded readiness status

Activation boundary: Use for software/package/release claims needing first-pass verification evidence; reviewer challenge follows rather than substitutes for it.

## verification-reviewer

- independently challenge a verification evidence chain
- test target fidelity, omissions, oracle strength, boundary realism, custody, traceability, authority, and decision fit
- issue review pass/conditions/fail and downgrade unsupported status

Activation boundary: Use only after verification evidence exists and present work will challenge it; future review assignment alone is route design.
