# Contributing

Useful contributions include reproducible tool defects, portability fixes, stronger discriminating tests, clearer host adapters, documented failure cases and corrections to unsupported claims.

Before opening a change:

1. describe the behavior and failure it prevents;
2. identify the affected package or testbed surface;
3. include a synthetic reproduction or exact evidence when possible;
4. keep unrelated cleanup out of the change;
5. run the narrowest relevant tests;
6. avoid credentials, private data and third-party proprietary material.

Opening an issue or pull request does not guarantee acceptance, support response time or inclusion in an official release. By submitting a contribution, you represent that you have authority to submit it and agree that any accepted contribution may be distributed under the license applicable to the part of the repository it modifies, unless Collaborative Dynamics agrees otherwise in writing.
