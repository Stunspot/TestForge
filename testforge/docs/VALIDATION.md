# Validation and evidence

TestForge separates package structure, tool execution, behavioral evaluation, host activation, and release authority.

Current release verification includes the current Builder profiles for the bundle, both Codex skill folders, both Claude-compatible folders, and both upload ZIPs; deterministic unit and distribution tests; package and manifest validation; evaluation-envelope validation; portable archive inspection; and customer ZIP round trip. The repository retains exact commands and results in its verification record.

Customer-document readiness is a separate gate. It requires an exact declared corpus, journey and navigation checks, accessible-Markdown source lint, a current content fingerprint, fresh exact-content Hesperos review, substantive evidence for all nine customer moments, zero unresolved material findings, and machine validation of the retained custody. Markdown lint alone cannot pass documentation readiness.

These checks do not establish defect freedom, live Codex or Claude activation, broad model reliability, production fitness, accessibility conformance, compliance, or release authorization. Every claim remains bounded to the version, environment, target revision, evidence cutoff, and executed paths recorded with it.
